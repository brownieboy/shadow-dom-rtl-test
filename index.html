Shadow DOM RTL Test

Language direction (sets dir attribute on DOM's body tag):


Plain HTML

Floating label's rtl works because global CSS has effect. Absolutely positioned left normally, or right when dir="rtl" is set on DOM's body tag.



Custom Element (No Shadow DOM)

Floating label's rtl works because global CSS has effect, same as for Plain HTML (see previous).




Web Component (Shadow DOM)

Floating label's rtl fails. It stays left because the global CSS has no effect in Shadow DOM. The CSS that is set here withing the Shadow DOM (i.e. [dir="rtl"]) also fails, because CSS inside a Shadow DOM cannot read the dir prop that was set outside of it.



Custom Web Component (Shadow DOM) Redesigned Label

Floating label's rtl works because it is not using absolute positioning for left or right. It has a wrapper that uses absolute positioning for top, but inside that, the browser takes care of rtl.
