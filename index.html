Shadow DOM RTL Test

Language direction (sets dir attribute on DOM's body tag):


Global CSS

Plain HTML

Floating label's rtl works because global CSS has effect. It's absolutely positioned left normally, or right when dir="rtl" is set on DOM's body tag.



Custom Element (No Shadow DOM)

Floating label's rtl works because global CSS still has effect when there's no Shadow DOM, same as for Plain HTML (see previous).




Web Component (Shadow DOM)

CSS Within Shadow DOM - Fails for RTL

Floating label's rtl fails. It stays left because the global CSS has no effect in Shadow DOM. Note that we're setting CSS inside the Shadow DOM to do the same thing but that fails too because CSS inside a Shadow DOM cannot see a dir prop that was set outside of it.



Custom Web Component (Shadow DOM) Redesigned Label

CSS Within Shadow DOM - Works!

Floating label's rtl works because it is not using absolute positioning for left or right any more. Instead, we use absolute positioning to set the top position only, and then the browser takes care of rtl automatically.
